What a message rule is

eDesk message rules trigger automatic actions that are applied to a ticket based on unique characteristics that you choose.

How you can use message rules on edesk

Some customers use message rules to automatically apply tags to messages or assign tickets to agents based on a variety of conditions, such as what marketplace the ticket comes from or what the associated product, customer or order value is. Others use it to assign incoming tickets evenly between customer service members in a round robin format. The options are many, and rules can be personalized to suit your business.

Where to create messge rules within edesk

Settings -> Smart tools -> +Message Rule

Glossary of Message Rule terms

Rule Settings

Name*: Give the rule a name. This is to reference the rule when it is applied in your account

Active: Tick to activate/deactivate a rule

Stop processing further rules if conditions are matched: If ticked no further rules in the sequence will be processed once this rule is applied

Skip logging in tickets when applied: A log will not be made that when this rule has applied to your ticket. Only use once per ticket: When a message matches the rule, the same rule will not be applied again

Rule Conditions

When creating the conditions under which you want the rules to be triggered, AND conditions must all be met, whereas with OR conditions only one of the conditions must be applied.

Rule Actions: Configure and select the action taken when the rule has been matched

Assigning to tags: Select the tag to be automatically assigned by the rule

Tag Assignation mode: If you have chosen folders above you can choose to assign to all of the folders you have selected or in a round robin format where it will select one folder at a time in the sequence each time this rule is matched

Assign ticket owner: Select the user to be automatically assigned as the owner

Assign to agent: Select a user to be mentioned on the ticket

Agent Assignation mode: If you have chosen agents above you can choose to assign to all of the agents you have selected or tp assign in a round robin format where it will select one agent at a time in the sequence each time this rule is matched

Change Ticket Status: Select Change ticket status if you would like the ticket status to change once the rule has been applied

Webhook URL: This allows you to send a notification to a URL once the rule has been applied
